新しい顧客がプラットフォームを使用開始する場合、Gmailアカウントでログインし、Googleグループを使用して、似たロールのチームメイトと共同作業をすることがよくあります
この方法は簡単ですが、チームのIDが一元管理されていないため、後で問題が発生することがあります
たとえばチームメンバーが退職した場合です
この設定ではチームのクラウドリソースに対するユーザーのアクセス権を、即座に削除する簡単な方法はありません
Cloud Identityというツールでは、Google管理コンソールからポリシーを定義しユーザーとグループを管理できます
管理者は、既存のActive Directory、またはLDAPシステムと同じ、ユーザー名とパスワードでログインし、Google Cloudリソースを管理できます
Cloud Identityを使用すると、退職者がいた場合に、管理者がGoogle管理コンソールからアカウントを無効にしグループから削除できます
Cloud IdentityにはFree Editionがありますが、Premium Editionではモバイルデバイスを管理できます
Google CloudとGoogle Workspaceを両方利用していれば、Google管理コンソールでこの機能を利用できます
