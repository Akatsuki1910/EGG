コンテナとKubernetesの基礎を学んだので、次にGKEを見ていきましょう
GKEは、Googleがホストするクラウド上のKubernetesマネージドサービスです
GKE環境は複数のマシン（Compute Engineインスタンス）で構成され、これらのマシンがグループ化されてクラスタを形成します
GKEでKubernetesクラスタを作成するには、Google Cloud Consoleを使うか、Cloud Software Development Kitに含まれるgcloudコマンドを使います
GKEクラスタはカスタマイズ可能で、さまざまなマシンタイプ、ノード数、ネットワーク設定をサポートします
クラスタの操作にはKubernetesのメカニズムを使います
Kubernetesのコマンドとリソースにより、アプリのデプロイと管理、管理タスクの実行、ポリシーの設定、デプロイされたワークロードの健全性のモニタを行えます
GKEクラスタを利用することで、Google Cloudの高度なクラスタ管理機能のメリットを活用できます
Compute Engineインスタンスで使用するGoogle Cloudの負荷分散、クラスタ内のノードのサブセットを定義するノードプールによる柔軟性の向上、クラスタノードのインスタンス数の自動スケーリング、クラスタノードソフトウェアの自動アップグレード、ノードの正常稼働と可用性を保つためのノードの自動修復、Google Cloudのオペレーションスイートでのロギングとモニタによるクラスタの可視化です
オンプレミスとクラウドのリソースを橋渡しする必要がある場合は、GKEクラスタでアプリを実行するのも良いでしょう
GKEでKubernetesクラスタの利用を始めるには、次のコマンドを実行するだけです
gcloud container clusters create k1
