最後にご紹介するストレージオプションは、Cloud Bigtableです
Cloud Bigtableはビッグデータ用のNoSQLデータベースサービスです
検索、アナリティクス、マップ、Gmailなど、Googleの多くのコアサービスを支えるデータベースでもあります
Bigtableは膨大なワークロードを一貫して低遅延、高スループットで処理できます
運用目的でも分析目的でも威力を発揮し、IoT、ユーザー分析、金融データ分析などに活用できます
Bigtableを選ぶべきケースは次のとおりです
１テラバイトを超える半構造化/構造化データを扱う場合
データが高スループットで高速移動、または急速に変化する場合
NoSQLデータを扱う場合
これは通常、強いリレーショナルセマンティクスが不要なトランザクションを指します
データが時系列または自然意味論的な順序の場合、ビッグデータを扱っていて、データに非同期バッチまたはリアルタイム同期処理を行う場合、機械学習アルゴリズムをデータに適用する場合です
Cloud Bigtableは他のGoogle Cloudサービスや外部クライアントとやり取りできます
APIを使えばデータサービスレイヤ経由で、Cloud Bigtableのデータを読み書きでき、データサービスレイヤにはマネージドVM、HBase Rest Serverのほか、HBaseクライアントを使うJava Serverなどがあります
通常これはアプリやダッシュボードなどにデータを提供するために使われます
またデータは、各種ストリーミング処理フレームワーク、Dataflow、Spark、Streaming、Stormなどでストリーミングできます
ストリーミングが選択肢にない場合は、Cloud Bigtableデータの読み書きを、Hadoop、MapReduce、Dataflow、Sparkなどのバッチ処理を通じて行えます
整理または新規計算されたデータは多くの場合、再びCloud Bigtable、または ダウンストリームDBに書き込まれます
