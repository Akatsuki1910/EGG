Google Cloudの第２のストレージオプションは、Cloud SQLです
Cloud SQLはフルマネージドリレーショナルデータベースで、MySQL、PostgreSQL、SQL Serverなどを利用できます
必要でありながら手間や時間がかかる作業をGoogleに移管できます
パッチや更新の適用やバックアップの管理、レプリケーションの構成などから解放されるので、優れたアプリの開発に専念できます
Cloud SQLではソフトウェアのインストールも保守も不要です
スケールアップは64個のプロセッサコア、400ギガバイト超のRAM、30テラバイトのストレージまで可能です
Cloud SQLのプライマリインスタンス、外部プライマリインスタンス、外部MySQLインスタンスなどからの自動レプリケーションをサポートしています
マネージドバックアップ対応なので、バックアップデータは安全に保管され、復元が必要なときにアクセスできます
インスタンス料金には7回分のバックアップが含まれます
Cloud SQLでは、データはGoogleのネットワーク上にあるときと、データベース、一時ファイル、バックアップへの保存時に暗号化されます
ネットワークファイアウォールで、各データベースインスタンスへのアクセスを制御できます
Cloud SQLインスタンスのメリットは、他のGoogle Cloudサービスや外部サービスからもアクセスできることです
Cloud SQLは、JavaのConnector/Jや、PythonのMySQLdbなどの、標準的なドライバーを使ってApp Engineと連携できます
Compute Engineインスタンスから、Cloud SQLインスタンスにアクセスしたり、Cloud SQLインスタンスをVMと同じゾーン内に構成したりできます
Cloud SQLは、SQL Workbench、Toad、MySQLドライバを使用する他の外部アプリなど、使用される可能性のあるその他のアプリやツールもサポートしています
