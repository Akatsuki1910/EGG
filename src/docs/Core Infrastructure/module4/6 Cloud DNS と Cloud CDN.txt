Googleの無料サービスの中でも有名なのが世界中で使えるDNSサービスの8.8.8.8です
DNSとはホスト名をアドレスに変換する仕組みです
ご想像のとおりGoogleは高度に発達したDNSインフラの持ち主です
8.8.8.8の提供はそのメリットを誰もが享受できるようにするためです
ではGoogle Cloudで構築したアプリのホスト名とアドレスはどうなるでしょうか
Cloud DNSを使えばアプリを世界に向けて公開できます
Googleと同じインフラで動作するマネージドDNSサービスです
低レイテンシで可用性が高く、コスト効率に優れた方法でアプリやサービスをユーザーに提供できます
公開したDNS情報は世界の複数の場所から冗長構成で提供されます
Cloud DNSはプログラムによる操作も可能です
数百万のDNSゾーンとレコードを、Cloud Console、コマンドライン、APIで公開、管理できます
世界規模のエッジキャッシュも使えます
キャッシュサーバーを使ってコンテンツをエンドユーザーの近くに格納します
このシステムを使ってCloud CDNでアプリでのコンテンツ提供を高速化できます
つまり、ユーザーが感じるネットワークレイテンシの低減、コンテンツ配信元の負荷の軽減、コスト削減が可能になります
HTTP負荷分散を設定したらワンクリックでCloud CDNを有効にできます
世の中には他にもさまざまなCDNがあります
別のCDNを使っていても、Google Cloud CDN Interconnectに含まれるものであればそのまま利用できます
