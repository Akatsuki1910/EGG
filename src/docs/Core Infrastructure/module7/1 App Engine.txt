本コースではこれまで、Google Cloudについて紹介し、Google Cloud上のVM、ネットワーク、ストレージ、コンテナの使用に関連する、オプションと利点を見てきました
このセクションでは、クラウドでのアプリ開発に目を向けます
最初に見るApp Engineは、ウェブアプリの開発とホスティングを大規模に行える、フルマネージドのサーバーレスプラットフォームです
その仕組みを説明します
App Engineでは、人気のあるプログラミング言語、ライブラリ、フレームワークから選んで、使い慣れたツールでアプリを開発した後、サーバーを自動的に用意して、アプリインスタンスをオンデマンドでスケールできます
つまり、コードをアップロードすれば、Googleがアプリの可用性を管理します
使用可能な開発ツールにはEclipse、IntelliJ、Maven、Git、Jenkins、PyCharmがあります
App Engineでは、サーバーを用意して維持する必要はありません
App Engineには各種サービスとAPIが組み込まれています
これにはNoSQLデータストア、Memcache、負荷分散、ヘルスチェック、アプリロギングのほか、ほとんどのアプリで使用されるユーザー認証APIが含まれます
App Engineはソフトウェア開発キット（SDK）を備えており、ローカルマシン上でのアプリの開発、デプロイ、管理に利用できます
各SDKに含まれているのはApp Engineで利用できるAPIとライブラリ、App Engineの全サービスをローカルでエミュレートするシミュレートされた安全なサンドボックス環境、アプリのクラウドへのアップロードやバージョン管理のためのデプロイツールです
SDKはアプリをローカルで管理するのに対し、Google Cloud Consoleは本番環境のアプリを管理します
Cloud Consoleのウェブベースのインターフェースを使い、新規アプリの作成、ドメイン名の設定、アプリのライブバージョンの変更、アクセスログやエラーログの確認など、さまざまな操作を行えます
セキュリティについては、Security Command CenterというGoogle Cloudのセキュリティ/リスク管理プラットフォームが、ウェブアプリの安全性を保ちます
Cloud ConsoleからSecurity Command Centerを使用して、ウェブアプリを自動的にスキャンし、一般的な脆弱性を検出できます
