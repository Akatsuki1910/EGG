Google Cloudのコースデベロッパーです
「Architecting with Compute Engine」の３コースを受講いただきありがとうございます
Google Cloud Platform（GCP）が提供するさまざまなサービスの使用を開始する前に、GCPがどのようなものかをご説明します
Google Cloudを見てみると、大規模なエコシステムの一部であることがわかります
このエコシステムはOSSプロバイダ、パートナー、デベロッパー、サードパーティソフトウェア、その他のクラウドプロバイダで構成されています
実はGoogleはOSSの強力なサポーター的存在なのです
そのとおりです
Google CloudはChrome、Googleデバイス、Googleマップ、Gmail、Googleアナリティクス、G Suite、Google検索、GCPで構成されています
GCP自体は次の主要機能を包括するコンピューティングソリューションのプラットフォームです
インフラ、プラットフォーム、ソフトウェアの３つです
このマップはGCPのグローバルインフラを表しています
本動画の作成時点で、GCPの十分にプロビジョンされたグローバルネットワークは60を超えるゾーンと130か所以上の拠点を光ファイバーケーブルで構築されたグローバルネットワークで繋いでいます
Googleはこのネットワークに継続的な投資を行うことで、新しいリージョン、拠点、海底ケーブルを増やしています
こうしたインフラの他にも、GCPは最先端のソフトウェア定義ネットワーキングと分散システム技術により、世界中でサービスをホストして提供しています
このような技術は拡張を続けるクラウドベースのプロダクトと、サービスに代表されます
多くのプロダクトとサービスは、この図にあるようなユニークな青い六角形のロゴで表されます
ここで重要なのはGCPでは、タスクやアプリのためのソリューションが１つだけではないということです
わかりやすいように、ソリューションを全体的に見てみましょう
GCPはIaaSからSaaSまで、幅広いサービスを提供しています
GCPでは支援や補助機能を提供するグローバルで自動スケーリング可能なウェブまたはモバイル向けアプリを構築できます
これらのアプリを通じて、ユーザーからインフラが見えないサービスを提供できます
Googleは検索、Gmail、Googleマップ、G Suiteなどのアプリの基盤となるインフラだけではなく、これらすべてのプロダクトをオープンにして、皆様が利用できるようパッケージ化しました
代替ソリューションも考えられます
たとえばCompute Engineで独自のVMを開始して、オープンソースのMySQLをインストールすれば、データセンターにある自社のコンピュータでMySQLデータベースのように実行できます
またはCloud SQLサービスを使って、MySQLインスタンスを提供し、バックアップやセキュリティパッチといった運用作業を処理させることもできます
この場合バックアップとパッチの自動化に、Googleと同じサービスを利用できます
自動スケーリングされるサーバーレスのNoSQLデータベースに移行することもできます
そうすれば成長に合わせて、サーバーインスタンスを追加したり、新たな容量に対処できるよう設計を変更したりする必要がありません
この一連のコースは、インフラに重点を置いています
ITインフラは都市のインフラに例えられます
このインフラは輸送、通信、電力、水道、燃料、その他の必要不可欠なサービスの基盤施設や、システムの基本フレームワークとなります
都市に住む人々はユーザーにあたり、車や自転車、建物はアプリのようなもので、これらのアプリをユーザー向けに作成し、サポートするために必要なものすべてがインフラとなります
本コースの目的は、GCPが提供するインフラサービスをできるだけ効率的にわかりやすく説明することです
本コースを通して、インフラサービスの理解を深めれば、サービスの機能と活用方法を把握できます
特定のカテゴリのアプリを使ったケーススタディの詳細には触れませんが、すべての構成要素から、独自のソリューションを構築できるようになります
GCPはさまざまなコンピューティングサービスを提供しています
最もわかりやすいサービスはCompute Engineかもしれません
Compute Engineではクラウド内でVMをオンデマンドで実行できます
Compute EngineはGoogle CloudのIaaSソリューションであり、サーバーインスタンスを自分で管理するユーザーにとっては、最も柔軟性の高い選択肢と言えます
GKEを使用するとコンテナ化されたアプリをGoogleが管理するクラウド環境で、管理、実行できます
コンテナ化は高いポータビリティと、効率的なリソース利用のために設計されたコードをパッケージ化する方法で、Kubernetesは複数のコンテナ内のコードをオーケストレーションする方法と言えます
App EngineはGCPで提供されるフルマネージドのPaaSフレームワークです
つまりインフラを意識することなく、クラウドでコードを実行できます
プロビジョニングとリソースの管理をすべてGoogleに任せて、コーディングに専念できます
App Engineの詳細は、「Developing Applications with Google Cloud Platform」で学ぶことができます
Cloud Functionsは完全にサーバーレスな実行環境であり、FaaSでもあります
イベントに基づいてコードを実行でき、１日に１回のイベントにも１秒に数回のイベントにも対応できます
Googleは必要に応じてリソースをスケールしますが、支払いはコードを実行している時間だけ発生します
「Developing Applications with Google Cloud Platform」でも Cloud Functionsについて説明しています
この一連のコースでは主に、Compute Engineについて説明します
「Architecting with Google Compute Engine」は、クラウドインフラ学習プログラムの一部です
クラウドにおけるアプリの実装、デプロイ、移行、保守を担当するITプロフェッショナルを対象としています
本講座の受講条件は、「Google Cloud Platform Fundamentals: Core Infrastructure」を修了していることです
この動画のリンクセクションをご覧ください
「Architecting with Google Compute Engine」は３つのコースで構成されています
「Essential Cloud Infrastructure: Foundation」は、その１つめのコースです
このコースでは最初にGCPの概要、GCP ConsoleとCloud Shellの操作方法を学びます
次に仮想ネットワークについて学び、VPCネットワークやその他のネットワーキングオブジェクトを作成します
そしてVMについて詳しく学んだ後で、Compute Engineを使用してVMを作成します
「Essential Cloud Infrastructure: Core Services」は、２つめのコースです
このコースでは、最初にCloud IAMについて確認した後、実際にIdentity and Access Managementでリソースを管理します
次にGCPのさまざまなデータストレージサービスを確認してから、その一部を実装します
その後リソース管理について学び、GCPリソースに対する請求の管理と確認を行います
最後にリソースのモニタリングについて学習し、StackdriverサービスでGCPリソースをモニタリングします
最後のコースは、「Elastic Cloud Infrastructure: Scaling and Automation」です
このコースでは最初に、ネットワークを相互接続するさまざまな方法を説明し、インフラをGCPに接続できるようにします
次にGCPのロードバランシングと自動スケーリングのサービスについて学び、これらのサービスを直接操作します
その後Deployment ManagerやTerraformなどのインフラ自動化サービスについて説明し、GCPインフラサービスの開発を自動化できるようにします
最後にGCPで活用できる、その他のマネージドサービスについて説明します
本講座の目標は、GCPのさまざまなサービスと機能を覚えて理解することと、その知識を活用して要件を分析し、さまざまなオプションを評価して独自のサービスを作成できるようになることです
そのため講座には、Qwiklabsプラットフォームのインタラクティブなハンズオンラボも含まれています
QwiklabsではGoogleアカウントと認証情報がプロビジョニングされるため、追加料金なしでGCP Consoleにアクセスできます
