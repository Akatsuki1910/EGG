Google Cloudコースデベロッパーです
「Architecting with Compute Engine」の３コースを受講いただきありがとうございます
Google Cloud Platform（GCP）が提供するさまざまなサービスの使用を開始する前に、GCPがどのようなものかをご説明します
Google Cloudは大規模なエコシステムの一部です
このエコシステムは、OSS、プロバイダ、パートナー、デベロッパー、サードパーティソフトウェア、その他のクラウドプロバイダで構成されます
実はGoogleはOSSの強力なサポーター的存在なのです
そのとおりです
現在Google CloudはChrome Googleデバイス、Googleマップ、Gmail、Googleアナリティクス、G Suite、Google検索、GCPで構成されています
GCP自体はコンピューティングソリューションのプラットフォームで、インフラ、プラットフォーム、ソフトウェアの３つの主要機能で構成されます
このマップはGCPのグローバルインフラを表しています
本動画の作成時点でGCPのグローバルネットワークは、光ファイバーケーブルで構築されたグローバルネットワークを通じ、60以上のゾーンと130以上の拠点をつないでいます
Googleはこのネットワークに継続的に投資し、新しいリージョン、拠点、海底ケーブルを増やしています
こうしたインフラのほかにも、GCPは最先端のソフトウェア定義ネットワーキングと、分散システム技術により、世界中でサービスをホストして提供しています
このような技術は拡張を続けるクラウドベースのプロダクトと、サービスに代表されます
ここで重要なのはGCPでは、タスクやアプリのためのソリューションが１つだけではないということです
わかりやすいようソリューションを全体的に見てみましょう
GCPはIaaSからSaaSまで、幅広いサービスを提供しています
GCPでは支援や補助を提供するグローバルで自動スケーリング可能なウェブ、モバイル向けアプリを構築できます
これらのアプリを通じて、ユーザーからはインフラが見えないサービスを提供できます
Googleは検索、Gmail、Googleマップ、G Suiteといったアプリの基盤となるインフラだけではなく、すべてのプロダクトをオープンにして皆様が利用できるようにパッケージ化しました
代替ソリューションも考えられます
たとえばCompute Engineで独自のVMを開始して、オープンソースのMySQLをインストールすれば、データセンターにある自社コンピュータ上のMySQLデータベースのように実行できます
またはCloud SQLサービスのMySQLインスタンスを使って、バックアップやセキュリティパッチといった運用作業を処理することもできます
この場合バックアップとパッチの自動化にGoogleと同じサービスを利用できます
自動スケーリングされるサーバーレスのNoSQLデータベースに移行することもでき、そうすれば成長に合わせたサーバーインスタンスの追加や、新たな容量に対処するための設計変更が不要になります
この一連のコースはインフラに重点を置いています
ITインフラは都市のインフラに例えられます
このインフラは輸送、通信、電力、水道、燃料、その他の必要不可欠なサービスの基盤施設や、システムの基本フレームワークとなります
都市に住む人々はユーザー、車や自転車、建物はアプリに該当し、これらのアプリをユーザー向けに作成し、サポートするために投入されるものすべてがインフラとなります
本コースの目的はGCPが提供するインフラサービスを、できるだけ効率的にわかりやすく説明することです
本コースを通してインフラサービスの理解を深めることで、サービスの機能と活用方法を理解できます
特定のカテゴリのアプリを使ったケーススタディの詳細には触れませんが、すべての構成要素から、独自のソリューションを構築できるようになります
GCPはさまざまなコンピューティングサービスを提供しています
最もわかりやすいサービスはCompute Engineです
Compute Engineではクラウド内でVMをオンデマンドで実行できます
Compute EngineはGoogle CloudのIaaSソリューションであり、サーバーインスタンスを自分で管理するユーザーにとって、最も柔軟性の高い選択肢と言えます
GKEを使用すると、Googleが管理するクラウド環境で、コンテナ化されたアプリを管理、実行できます
コンテナ化はポータビリティと、効率的なリソース利用のために設計されたコードをパッケージ化する方法で、Kubernetesは複数のコンテナ内のコードをオーケストレーションする方法と言えます
App EngineはGCPで提供されるフルマネージドのPaaSフレームワークです
つまりインフラを意識することなくクラウドでコードを実行できます
プロビジョニングとリソースの管理をすべてGoogleに任せて、コーディングに専念できます
App Engineの詳細は、「Developing Applications with Google Cloud」で学ぶことができます
Cloud Functionsは完全にサーバーレスな実行環境であり、FaaSでもあります
イベントに基づいてコードを実行でき、１日１回のイベントにも毎秒数回のイベントにも対応できます
Googleは必要に応じてリソースをスケールしますが、支払い対象はコードの実行時間分のみです
「Developing Applications with Google Cloud Core」でも、Cloud Functionsについて説明します
Cloud Runではウェブリクエスト、またはPub/Subイベントを通じて、ステートレスコンテナを実行できます
Cloud Runはサーバーレスなのでインフラ管理が一切不要で、アプリの開発に集中できます
また、Knativeを基盤とし、Kubernetes上のオープンなAPI、ランタイム環境を利用するため、ワークロードを複数の環境とプラットフォームの間で移動でき、Google Cloud、GKEまたはKnativeのあらゆる実行環境でフルマネージドで運用できます
Cloud Runは高速で、スケールアップ、ダウンをゼロから自動で瞬時に行います
課金対象は使用したリソース分のみで、100ミリ秒未満は切り捨てられます
オーバープロビジョニングされたリソース分の料金は請求されません
本コースではCompute Engineを中心に説明します
「Architecting with Google Compute Engine」は、クラウドインフラ学習プログラムの一部です
クラウドにおけるアプリの実装、デプロイ、移行、保守を担当する、ITプロフェッショナルを対象としています
本コースの受講条件は、「Google Cloud Platform Fundamentals: Core Infrastructure」を修了していることです

「Architecting with Google Compute Engine」は、３つのコースで構成されています
「Essential Cloud Infrastructure: Foundation」は、その１つ目のコースです
このコースではまずGCPの概要、GCP ConsoleとCloud Shellの操作方法を学びます
次に仮想ネットワークについて学び、VPCネットワークなどのネットワーキングオブジェクトを作成します
そしてVMについて詳しく学んだ後、Compute EngineでVMを作成します
「Essential Cloud Infrastructure: Core Services」は、２つ目のコースです
このコースでは、Cloud IAMについて確認した後、実際にIdentity and Access Managementでリソースを管理します
次にGCPの各種データストレージサービスを確認してから、その一部を実装します
その後リソース管理について学び、GCPリソースに対する請求の管理と確認を行います
最後にリソースのモニタリングについて学習し、StackdriverサービスでGCPリソースをモニタリングします
最後のコースは、「Elastic Cloud Infrastructure: Scaling and Automation」です
このコースでは最初に、ネットワークを相互接続するさまざまな方法を説明し、インフラをGCPに接続できるようにします
次にGCPのロードバランシングと自動スケーリングサービスについて学び、これらのサービスを直接操作します
その後Terraformなどのインフラ自動化サービスについて説明し、GCPインフラサービスの開発を自動化できるようにします
最後にGCPで活用できる他のマネージドサービスについて説明します
本講座の目標は、GCPのさまざまなサービスと機能を覚えて理解し、その知識を活用して要件を分析し、さまざまなオプションを評価して、独自のサービスを作成できるようになることです
そのため、Qwiklabsプラットフォームのインタラクティブなハンズオンラボもあります
QwiklabsではGoogleアカウントと認証情報がプロビジョニングされるため、追加料金なしでGCPコンソールにアクセスできます
