プロジェクトとフォルダによって、リソースを分離できますが
さらに細かく分ける場合はラベルを使います
ラベルはGoogle Cloudリソースを整理するためのユーティリティで、VM、ディスク、スナップショット、イメージなどのリソースに設定できるKey-Valueペアです
Google Cloud Console、gcloud Resource Manager APIを使用して、ラベルを作成、管理できます
各リソースには最大64のラベルを設定できます
たとえば仮想マシンの環境を定義するラベルを作成する場合、各インスタンスに本番またはテストのいずれかのラベルを定義します
このラベルを使用して、すべての本番環境リソースを検索して管理用にリストできます
ラベルをスクリプトで使用すれば、費用の分析や複数のリソースに対する一括操作の実行も可能です
右側の図は１つのインスタンスに作成された４個のラベルの例を示しています
ラベルの使用例をいくつか見てみましょう
チームまたはコストセンターに基づくラベルを追加して、それぞれのチームが所有するインスタンスを区別することをおすすめします
この種類のラベルは、費用計算や予算作成に使用できます
たとえば、team:marketingやteam:researchなどです
ラベルを使用して、コンポーネントを区別することもできます
たとえば、component:redisやcomponent:frontendです
環境またはステージに基づくラベルも使用できます
ラベルでリソースの所有者またはメインの連絡先を定義することも検討してください
たとえば、owner:gauravやcontact:opmなどです
または、リソースの状態を定義するラベルを追加します
たとえば、state:inuseやstate:readyfordeletionなどです
ラベルとタグを混同しないことが重要です
ここで学んだようにラベルは、ユーザーが定義するKey-Value形式の文字列です
リソースの整理に使用され、課金に反映されます
一方タグは、インスタンスのみに適用されるユーザー定義の文字列です
主にファイアウォールルールの適用といった、ネットワーキングに使用されます
ラベルの使用について詳しくは、この動画のリンクセクションをご覧ください
