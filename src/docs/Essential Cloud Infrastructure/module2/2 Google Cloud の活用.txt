GCPを操作するための４つの方法を順に紹介します
Google Cloud Platform Console（GCP Console）、Cloud ShellとCloud SDK API、Cloudモバイルアプリの４つです
GCP Consoleでは、ウェブベースのGUIを使用できます
これにはconsole.cloud.google.comからアクセスします
たとえば、上部に示すように仮想マシンとその詳細を確認できます
ターミナルウィンドウがお好みの場合は、Cloud SDKのgcloudコマンドラインツールを使用できます
たとえば、下のように仮想マシンとその詳細を一覧表示するには、"gcloud compute instances list"コマンドを使用します
GCPではブラウザベースの対話型シェル環境としてCloud Shellも使用できます
これにはGCP Consoleからアクセスできます
Cloud Shellは５GBの永続ディスクストレージを持つ一時的な仮想マシンで、Cloud SDKがプリインストールされています
このコースでは、さまざまなラボで学習した内容を応用します
ラボにはGCP Consoleを使用する手順が含まれています
“メニューで[Compute Engine]>[VMインスタンス]をクリック”といった具合です
手順の実施方法を説明します
まず、GCP Consoleで３本の横線のアイコンをクリックします
これはナビゲーションメニューで画面の左側にあります
クリックすると、右のようにメニューが開き、主要なプロダクトとサービスがすべてこのメニューに一覧されます
メニューで[Compute Engine]にカーソルを合わせてサブメニューを開きます
最後にサブメニューで[VMインスタンス]をクリックします
ラボを実施していくうちに、このような手順とGCP Consoleの操作に慣れてきます
ラボにはコマンドラインを使用した手順もあります
操作はコマンドをコピーして、Cloud ShellかSSHターミナルに貼り付けるだけですが、コマンドの変更が必要になる場合もあります
グローバルに一意な名前をStorageバケットに付けるときなどです
Cloud SDK に加えて、リソースを簡単に作成、管理できるクライアントライブラリも利用できます
GCPクライアントライブラリでは２つのAPIを公開しています
サービスにアクセスするためのアプリAPIは、Node.jsやPythonなどのサポート対象の言語用に最適化されています
管理APIはリソースの管理機能を提供し、独自の自動化ツールを構築する場合などに使用できます
CloudモバイルアプリもGCPの操作方法の一つで、AndroidまたはiOSデバイスからGCPサービスを管理できます
たとえばCompute Engineインスタンスの起動、停止、SSH接続や、各インスタンスのログの表示を行えます
また、カスタマイズ可能なグラフを設定し主な指標としてCPU使用率、ネットワーク使用率、リクエスト数、サーバーエラーなどを表示できます
アラートやインシデント管理の機能もあるので、プロジェクトの最新のお支払い情報を取得したり、予算を超過しているプロジェクトのアラートを受けたりできます
CloudモバイルアプリはGoogle PlayまたはApp Storeからダウンロードできます
