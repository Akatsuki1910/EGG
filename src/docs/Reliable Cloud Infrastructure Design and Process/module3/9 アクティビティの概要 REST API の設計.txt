ここでは、自分のアプリ向けに特定したマイクロサービスのAPIを設計します
このアクティビティの目的は、実際にAPIを設計して経験を積むために、API URL構造メッセージのリクエストとレスポンスの形式、バージョニングなどの側面を検討することです
たとえばオンラインストア向けのAPIを定義するとします
ここに示すようなAPIの定義が考えられます
サービスでUIやバックエンドの機能を提供するかどうかを問わず、HTTPSで呼び出せるプログラマティックインターフェースが必要です
ウェブサイトとウェブサービスの違いは、返されるデータの形式だけです
UIサービスに対してはHTMLが返されますが、バックエンドからはJSONまたはXMLが返されることもあります
このモジュールで学習した原則を用いるとともに、設計ワークブックのアクティビティ５を参照してください
